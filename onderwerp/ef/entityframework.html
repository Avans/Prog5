Entity Framework

Het aansluiten van een database is bijna altijd het zelfde werk. Objecten maken die tabellen in je database reflecteren en queryâ€™s uitvoeren om daarna de data uit de database te mappen op je objecten. "Entity Framework" is een Object Relational Mapper (ORM) die dit voor je oplost. Het zorgt er voor dat je maar een keer je data-model hoeft te defineren (code of database) . De rest zal "Entity Framework" voor je genereren.

Werkwijze

Als je met Entity Framework aan de slag gaat zijn er 3 manieren om met het framework te werken. Wij gaan voornamelijk inzoomen op de werkwijze Database first.

  • Model First
  • Code First
  • Database First

Als je een bestaande database hebt is dit ideaal. Op basis van een bestaande database zal Entity Framework een volledig data-model generen. Deze functie werkt heel goed. Daarnaast is het praktisch om wijzigingen door te voeren vanuit SQL server management studio, en nadat je klaar bent met je data-model, deze wijzigingen ook door te voeren in je applicatie.

Entity Framework biedt een mooie tool voor het bewerken van je data-model. Als je geen fan bent van het werken in SQL management studio dan is dit een prettige manier van het bewerken van je data-model. Daarentegen heb je geen volledige toegang tot je data-model aangezien deze gegenereerd wordt door een tool.

Het geeft je volledig toegang tot je entiteiten (modellen) en de code er achter. Op basis van deze code wordt er een database gegenereerd. In principe is dit een goed systeem wanneer je kleine hoeveelheden data nodig hebt en de database alleen voor dat programma nodig is.

Installatie

Entity framework is een framework dat niet standaard in .NET wordt meegeleverd. Dit betekend dat je hem zelf zal moeten installeren. Dit kan zeer gemakkelijk via de NuGet wizard. Dit hebben we eerder al een keer behandeld tijdens het onderdeel MvvM Frameworks.

Onderdelen

Na het installeren van het Entity Framework krijg je als ontwikkelaar toegang tot een aantal componenten. Hieronder volgt een opsomming van de elementen waar we het meeste mee aan de slag gaat. Laten we er voor het gemak even vanuit gaan dat we al een database hebben om mee aan de slag te werken!

EDMX file

Het belangrijkste component van het Entity Framework. EDMX staat voor Entity Data Model XML.

De EDMX file is een bestand gebaseerd op XML, een soort markup taal zoals HTML. Je hoeft XML nog niet te kennen, aangezien we dit bestand niet met de hand gaan aanpassen. Entity Framework heeft namelijk een mooie editor en handige Wizards om dit bestand te doen.

Het aanmaken van de EDMX file doen we via de Entity Data Model Wizard en het aanpassen van de file doen we via de Entity Designer.

In het volgende hoofdstuk gaan we zien hoe we de EDMX file kunnen genereren op basis van een bestaande database.

Data Context

Na het genereren van de EDMX file krijgen we toegang tot een aantal klasse. Een van deze klasse is je toegang tot de database. Het is een klasse die overerft van de klasse DbContext.

Deze klasse wordt gegenereerd door Entity Framework en we mogen deze klasse dus ook niet met de hand aanpassen! Hoe we dan wel aanpassing doorvoeren bekijken we in het volgende hoofdstuk. 

public partial class DatabaseContext : DbContext
{
    public DatabaseContext()
        : base("name=DatabaseContext")
    {
    }

    protected override void OnModelCreating(DbModelBuilder modelBuilder)
    {
        throw new UnintentionalCodeFirstException();
    }

    public virtual DbSet<C__RefactorLog> C__RefactorLog { get; set; }
    public virtual DbSet<Competition> Competitions { get; set; }
    public virtual DbSet<Team> Teams { get; set; }
}

DbSet

Deze context klasse bevat voor elke tabel in je database een property van het type DbSet. Dit is een lijst van elementen van een bepaalde entiteit.
