Entity Framework

Database first

De onderdelen van het Entity Framework zijn nu in kaart gebracht. Hoe passen we deze onderdelen toe in de praktijk?

Er zijn dus 3 verschillende manieren om met Entity Framework te werken. Wij gaan de Database first werkwijze toepassen. Waarom?

  • Een data architect kan de database ontwerpen
  • Je database design staat los van Entity Framework en .NET
  • Extra voordelen van een Database project (publish scripts, etc..)

De eerste stap in het werken met Entity Framework is dus het maken van de Database! Gelukkig hebben we dit al behandeld. Wat moet er nog wel gebeuren?

  1. EDMX genereren
  2. Gegevens ophalen uit de database.
  3. Gegevens opslaan in de database.
  4. Wijzigingen doorvoeren in de database.

1. EDMX Genereren

Entity Framework geeft ons een mooie wizard om een EDMX file toe te voegen op basis van een bestaande database. Hier onder staat een korte screencast met daarin de stappen om een EDMX file te genereren.

2. Gegevens ophalen

Nu we onze EDMX hebben gegenereerd hebben we toegang tot een aantal klasse. Deze klasse gaan we gebruiken om gegevens op te halen. Kijk eens goed naar de onderstaande code snippet. 

//Summary van alle workshops
using (var context = new Entities.Domain.MyEntities())
{
    //Niet vergeten de 'teacher' ook op te vragen!
    List<Workshop> workshops = context.Workshops.Include("Teacher").ToList();
}

In deze snippet maken we gebruik van 2 van de gegenereerde klasse. We gebruiken de klasse DataContext om een lijst van de workshops op te halen. Tot nu toe doen we nog niks met deze lijst behalve hem in een collectie van het type List te zetten.

De 'using' clausule is een speciale syntax binnen C#. Aangezien we stiekem onderwater verbinding maken met een database, zal de connectie met deze database ook een keer gesloten moeten worden. De 'using' syntax lost dit probleem voor ons op. Aan het einde van de 'using' clausule wordt de verbinding met de database automatisch verbroken.

De include methode staat er ook niet voor niets. Deze code snippet komt uit de workshop opdracht. In deze opdracht staat een bijbehorend database project waar je kan zien dat een workshop een foreign key heeft naar de tabel teacher. Entity Framework herkent deze foreign keys en maakt een zogenaamd 'navigation property' aan. Dit houd in dat je (als je tijdens het ophalen van de data de include methode aanroept) ook meteen de bijbehorende teacher krijgt!

3. Gegevens opslaan

Gegevens opslaan kost met Entity Framework bijna geen moeite! Maak een object van de klasse die je wilt toevoegen. Voeg hem toe aan de juiste lijst in de context. De wijziging wordt pas doorgevoerd als je 'SaveChanges' aanroept op de context!

Het kan zijn dat het opslaan van de gegevens fout gaat. Bijvoorbeeld als je niet alle velden van het object goed hebt ingevuld. In dat geval zal Entity Framework een Exception geven na het aanroepen van 'SaveChanges'. 

Workshop ws = new Workshop()
{
    Name = "Ei bakken",
    Summary = "Leer hoe je een ei bakt",
    TeacherId = 1,
};

context.Workshops.Add(ws);
//Altijd save changes aanroepen na een wijziging!
context.SaveChanges();

4. Wijziging doorvoeren

Het doorvoeren van een wijziging begint bij het aanpassen van het Database project. Hierna moet je de EDMX file updaten en klaar is kees! Over het doorvoeren van een wijziging is ook een korte screencast beschikbaar.
