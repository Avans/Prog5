Lambda expressies

"A lambda expression is an anonymous function that you can use to create delegates or expression tree types. By using lambda expressions, you can write local functions that can be passed as arguments or returned as the value of function calls. Lambda expressions are particularly helpful for writing LINQ query expressions.""

Duidelijk toch? Dat dacht ik al

lambda expressies worden gebruikt om Linq queries (zie volgende blokje) te schrijven en om deze toe te kunnen passen is de basiskennis nodig. Laten we beginnen met de vorm van een lambda expressie. 

  n => n % 2 == 1

Wat doet deze expressie nu eigenlijk? Als je een lambda expressie definiëert, definiëer je onder water een 'anonieme' functie. Dit is een functie zonder naam, die na het uitvoeren wordt weggegooid. Lambda expressies zijn niet meer dan een 'korte syntax' om snel even een functie te definiëren en uit te voeren. De bovenstaande lambda expressie bestaat uit de volgende onderdelen:

  • n is de input parameter (parameters van de functie)
  • n % 2 == 1 is de expressie (inhoud van de functie)

Deze expressie vertelt ons niet zo veel. Maar laten we nu een wat realistischer scenario nemen. Stel, je wilt een lijst filteren. Normaal zou je dan een for loop definiëren, door alle elementen van je collectie loopen en je filter (eventueel functie) toepassen.

Een lambda expressie kan dit voor ons een stuk simpeler maken.

Old scool

 
public void DoSomething()
{
    List<Person> lijstje = new List<Person>();
    List<Person> gefilterd = new List<Person>();
    
    gefilterd = lijstje.Where(LastnameSmulders).ToList();
}

public bool LastnameSmulders(Person person)
{
    return person.Lastname == "Smulders";
}

Lambda

 
List<Person> lijstje = new List<Person>();
List<Person> gefilterd = new List<Person>();

gefilterd = lijstje.Where(p => p.Lastname == "Smulders")
  .ToList();

De 'Where' functie op de lijst is onderdeel van Linq. Hier gaan we het zo over hebben. Deze vervangt voor ons de 'for loop'.

Onze lambda statement vervangt in dit voorbeeld de functie "LastnameSmulders". de p die we meegeven is de naam van de parameter die onze 'anonieme functie' meekrijgt. De 'Where' statement roept onze anonieme functie aan met elke keer 1 persoon uit de lijst. Onze expressie wordt omgezet in een boolean. Je zou dit dus als een 'if' statement kunnen lezen. Alle personen waarbij deze expressie 'true' is worden verzameld in een nieuwe lijst als return value van de functie Where.

Als je vaker gebruik gaat maken van lambda expressies, voelt de syntax vanzelf vertrouwd aan. lambda expressies kunnen je een hoop tijd besparen!

Het meegeven van een functie als 'parameter' valt onder de categorie 'delegates'. Hier gaan we later dit blok nog op in.
