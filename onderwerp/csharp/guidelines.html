Coding Guidelines

Coding Guidelines houdt in dat we afspraken gaan maken over hoe onze code 'stylen' Hierbij moet je denken aan bijvoorbeeld de volgende dingen:

  • Inspringen
  • Haakjes
  • Wanneer een lege regel
  • Wanneer & hoe commentaar

Het maakt niet uit welke coding guidelines je kiest als iedereen binnen je team maar dezelfde guidelines hanteert. Bijvoorbeeld de microsoft coding guidelines.

https://msdn.microsoft.com/en-us/library/ff926074.aspx

Daarnaast is het belangrijk om je te houden aan de coding conventions. Coding conventions zijn afspraken over de naamgeving binnen C#. In ons geval zijn dit PascalCasing en camelCasing.

PascalCasing ( NumberOfWheels )

PascalCasing houdt in dat we de naam van een variabele beginnen met een hoofdletter. Verder beginnen alle lossen woorden binnen de naam ook met een hoofdletter. PascalCasing passen we toe op de volgende onderdelen:

  • Classes
  • Namespaces
  • Methods
  • Constructors / Deconstructors
  • Properties

camelCasing ( numberOfWheels )

camelCasing is ongeveerd het zelfde als PascalCasing, op het feit na dat de eerste letter van de naam een kleine letter is. camelCasing passen we toe op de volgende onderdelen:

  • Fields
  • Lokale variabelen
  • Method parameters
