Assemblies

Vanuit Java ben je als het goed is al gewend om gebruik te maken van namespaces. Dit is een soort van 'achternaam' van een klasse. Het kan zo maar zijn dat een naam van een klasse 2x voorkomt in een applicatie. Namespaces zorgen er voor dat deze klasse nog steeds uniek zijn.

In Java kun je gebruik maken van een namespace via het Import statement. Binnen .NET doe je dit met het Using statement.

Java vs .NET

In .NET gaan we nog een laagje dieper. Klassen worden opgedeeld in namespaces, en namespaces zijn weer onderdeel van een library of assembly. In java krijg je een grote JDK (Java Development Kit). Hierin zitten heel veel klassen waar je gebruik van kan maken. Dit is niet het geval in .NET. Hier worden kleinere sets van klassen opgeleverd in assemblies. Deze assemblies vind je terug in je project in de vorm van DLL's. (Dynamic Link Library).

In elk project in .NET kun je referenties (references) leggen naar andere libraries. Deze libraries bevatten namespaces, en deze namespaces bevatten klassen. Elke template die je start binnen Visual Studio bevat een andere set aan standaard assemblies. Naast deze assemblies kun je zelf nog meer assemblies toevoegen. Deze assemblies kunnen zich al op je computer bevinden omdat ze bijvoorbeeld onderdeel zijn van het .NET framework, of komen online van een externe bron.
