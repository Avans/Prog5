WPF

Introductie

WPF is een afkorting voor Windows Presentation Foundation. Dit Framework biedt je de tools om een rijke user interface te bouwen. Dit is niet de enige unieke eigenschap van WPF. De mogelijkheid tot het data-binden van je view aan je model geeft je een beter scheiding tussen je GUI en je bedrijfslogica.

Naast de onderliggende structuur, die je helpt een nette en onderhoudbare interface te bouwen, ondersteunt WPF ook meerdere mediaformaten. Een van deze formaten is XAML, een markup language die gebaseerd is op XML*. Maar denk hierbij ook aan 2-D en 3-D Graphics, animaties, losse stylesheets, templates, documenten en media files (mp3, mp4).

*XML (Extensible Markup Language) Een markup language die regels en encoding voor een document vastlegt, net als HTML.

WPF is resolutieonafhankelijk. Dit houdt in dat op verschillende resoluties hetzelfde resultaat getoond zal worden. De XAML interfaces worden vector-based gerenderd.

Interface design

WPF biedt je de mogelijkheid om je interfaces op 2 manieren te ontwikkelen. Het definiëren van je GUI in XAML of het opbouwen van je GUI in code ('code-behind').

XAML

 
<Window x:Class="XAMLCodeBehind.MainWindow"
	xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
	xmlns:x="http://schemas.microsoft.com/winfx/2006/xaml"
	Title="MainWindow" Height="350" Width="525">
	
	<Grid>
		<Button>Click Me</Button>
	</Grid>
</Window>

Code behind

 
public partial class MainWindow : Window
{
	public MainWindow()
	{
		InitializeComponent();

		Button button = new Button();
		button.Content = "Click me!";
		Griddy.Children.Add(button);
	}
}

Het is netjes en onderhoudbaar om het uiterlijk van de applicatie volledig in XAML te definiëren. Het gedrag van de applicatie kan in de 'code-behind' van de applicatie worden gedefinieerd.

Het design van je applicatie heeft op deze manier een losse koppeling met het gedrag van de applicatie. Hierdoor kan een designer los aan de GUI werken terwijl een ontwikkelaar het gedrag van de applicatie tot stand brengt. Er zijn ook verschillende tools om aan de GUI te werken. Een van deze tools is Microsoft Blend.

Je kunt nu vanuit de code-behind file bij de XAML elementen die hebt je gedefinieerd. De waardes die je op je model (bijv. properties) hebt staan kun je nu dus mee geven aan deze elementen. Het is daar en tegen netter om deze informatie niet te 'setten' maar te 'data-binden'.
