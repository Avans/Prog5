WPF

XAML

De opbouw van een XAML document is vergelijkbaar met een HTML document. Je bouwt een boom van Elementen en de eigenschappen van deze elementen plaats je in Attributen. Deze geneste objecten leiden tot een 'element tree' die gecompileerd wordt tot een user interface.

XAML is een afkorting voor XML based Markup language. Even voor de duidelijkheid, je spreek het uit als 'zammel'. Het is een onderdeel van het WPF platform. Je kunt XAML definiëren in een .XAML file net als in een .HTML file. Elke XAML file heeft meestal een bijbehorende .CS file waar de 'code-behind' staat, maar dit is niet verplicht. De onderstaande technieken ondersteunen XAML.

  • Windows Store apps
  • Windows Phone apps
  • Silverlight
  • WPF

Wij gaan het voornamelijk inzetten voor WPF.

Je kunt de volgende elementen met XAML definiëren:

  • Windows
  • dialog boxes
  • pages
  • user controls!
  • controls
  • Shapes
  • graphics

XAML

 
<Window
  xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation"
  Title="Window with Button"
  Width="250" Height="100">

	<!-- Add button to window -->
	<Button Name="button">Click Me!</Button>

</Window>

WPF wordt voornamelijk gebruikt voor desktop-applicaties en Windows Apps. Daarnaast heb je ook nog browser-hosted-applicaties. Deze applicaties hebben de naam XBAP'S. Wij gaan voornamelijk aan de slag met de desktop applicaties.

Je kunt een volledige applicatie opbouwen met verschillende windows en pages. Je kunt zoveel elementen toevoegen als je zelf wilt. Je bent daarentegen wel verplicht tot het gebruiken van een Application element.

In een console-applicatie heb je een Program class met een Main functie. Dit is de plek waar je applicatie begint. Een dergelijke class heb je ook in elementen met XAML. Dit is de Application class en heeft een bijbehorend XAML element wat hem tot stand brengt. In dit element kun je aangeven bij welk window je wilt beginnen.

Console application

//console app
class Program
{
    static void Main(string[] args)
    {
    }
}

WPF application


<Application
    xmlns="http://schemas.microsoft.com/winfx/2006/xaml/presentation" 
    StartupUri="MainWindow.xaml" />    
public partial class App : Application
{
	//code behind logic
}

De aangeleverde MainWindow.xaml file bevat al standaard een titel, breedte, hoogte en Grid als child element. Nu kunnen we zelf een elementenboom gaan opbouwen in deze grid om een volledige user interface te krijgen. Deze boom bestaat uit XAML elementen.

Elk XAML element bevat een bijbehorende referentie naar een .NET class. Je kunt dus ook zelf een class omzetten naar een XAML element. De meest voorkomende manier waarop dit gebeurt zijn user controls. Het .NET framework bevat al een grote lijst aan voorgedefinieerde classes.
