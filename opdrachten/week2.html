Welkom in CSharp City. Een stad vol mogelijkheden en valkuilen.

Als nieuwe hacker in de stad wil je je skills testen op het allerhoogste niveau. Inbreken in een bank

Maak gebruik van je kennis van C# om de kluis te openen en het geld te stelen. Om dit te doen zul je een aantal 'hacks' moeten uitvoeren.

Je gaat aan de slag in de klasse 'Thief'. Hier vind je de volgende methodes die je zelf zult moeten uitbreiden.

OpenGate()

De guard heeft een (private) sleutel. Voeg een public property toe aan de klasse Guard waarmee je de sleutel kan ophalen.

OpenVault()

Bekijk de klasse Vault. Is er een manier om het private field '_isOpen' op true te krijgen?

DodgeLazors()

De meeste hackers zijn niet echt atletisch, dus laten we de "lazors" uitschakelen.

Gebruik een Linq query om elke lazors te 'disablen'.

Start project

Who you gonna call?!

In deze oefening ga je een eenvoudig telefoonboek maken in .NET. Het doel van deze oefening is te leren omgaan met C# properties en LINQ expressies.

Net als in de voorgaande oefening (1.1) is het van belang om netjes en gestructureerd te coderen. Houd je daarbij ook aan de C# Naming Conventions. Maak een class Person. De class moet de volgende properties bevatten: FirstName, LastName en TelephoneNumber. Zorg daarnaast voor een Read-Only property FullName die de combinatie FirstName en LastName terug geeft. Zorg ook voor een constructor waarmee je een firstname & lastname mee kan geven.

Maak een class TelephoneBook deze bevat intern een collectie van Person objecten Tip: gebruik hiervoor List. Vul deze list hard-coded met gegevens van minimaal 10 personen. De TelephoneBook class moet de volgende functionaliteiten hebben. implementeer deze functionaliteiten zoveel mogelijk met LINQ expressies.

  • Een methode die alle personen uit het telefoonboek gesorteerd op achternaam terug geeft.
  • Een methode die alle personen uit het telefoonboek terug geeft waarvan de voornaam begint met een opgegeven letter. Deze lijst moet ook weer gesorteerd op achternaam zijn.
  • Een methode die alle achternamen van personen uit het telefoonboek terug geeft waarvan de lengte van de achternaam groter is dan een opgegeven waarde.
  • De lijst van achternamen moet gesorteerd op lengte teruggegeven worden waarbij de langste achternaam onderaan staat en de kortste achternaam bovenaan.
  • Een read-only property die aangeeft hoeveel personen er in het telefoonboek zitten.

Test je code door de functies van je telefoonboek aan te roepen en het resultaat te tonen in een Console.
