CRU-Delete

De Create en Read actie van de CRUD werken. De volgende stap is het implementeren van een Delete actie. Zorg ervoor dat de MusicCollection applicatie deze functionaliteit ook ondersteunt. De delete knop is al aanwezig, maar jij moet er voor zorgen dat deze ook een liedje uit de lijst verwijderd.

Tip: Maak gebruik van een RelayCommand

Tweede View

Voeg een tweede View toe aan je applicatie waarin je met een vorige en volgende knop door de collectie van je CRUD applicatie heen kan bladeren. Maak gebruik van hetzelfde View Model en Model.

Kijk goed hoe de MainWindow aan zijn data context komt.

CR-Update-D

De Create en Read actie van onze CRUD werken. Nu moet de Update actie nog geïmplementeerd worden. Zorg ervoor dat de 'edit' knopt werkt.

Als een gebruiker op de 'edit' knopt klikt moet er een nieuwe window openen waarmee de gebruiker een song kan aanpassen.

Tip: Maak slim gebruik van de property SelectedSong

Deze week geen start project. De link aan het begin geeft je toegang tot een github classroom repository. Je zult zelf alle onderdelen samen moeten stellen om een werkende applicatie te krijgen. Kijk goed naar de uitwerking van de workshop opdracht als je er even niet meer uit komt.

Inventory

Bouw een applicatie waarbij een lijst van items zichtbaar is. Deze lijst mag gewoon hard-coded in een klasse staan (zie Dummy Repository workshop opdracht). Het moet mogelijk zijn om een van de items te selecteren en deze toe te voegen aan de inventory.

De inventory moet zichtbaar zijn in een apparte window. Er mogen maximaal 6 items aan de inventory toegevoegd worden.

Eisen

  • Er moeten minstens 10 items in de lijst staan
  • Als een item in de inventory zit moet hij uit de lijst verwijders worden
  • Als de inventory vol zit moet de knop voor het toevoegen uitgeschakeld worden

Technische eisen

  • Maak gebruik van het MvvM Light framework
  • De applicatie moet voldoen aan de eisen van MvvM
  • De inventory en lijst moeten in 2 apparte windows staan
