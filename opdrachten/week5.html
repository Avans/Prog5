Voordat we beginnen

Omdat we gaan werken met een database moet je deze zelf eerst deployen voordat je aan de slag kan.

In het start project zit een project met de naam Entitiese.Database. Hier staat een 'publish' file waar je gebruik van kan maken. Publish de database op je lokale server (localhost of localdb).

Indien je niet op de standaard localdb server gaat werken zijn er nog wat extra handelingen nodig. De projecten Entities.Domain en Entities.Console hebbene beide een configuratie file met een connecion string. Deze verwijzen naar de standaard localdb server. Pas deze connection strings aan zodat ze naar de juiste server verwijzen.

Start project

Klaar met de workshop opdracht? Dan gaan we nu Entity Framework toepassen in een WPF project!

In het start project staan 3 projecten

  • LeagueOfEntities.Database
  • LeagueOfEntities.Domain
  • LeagueOfEntities.WPF

Voordat je kan beginnen moet je de database deployen en controlleren of de connection strings nog goed staan!

De applicatie is al zover af dat wanneer je deze start er een lijst van competities met elk 2 teams beschikbaar is.

De opdracht

Een team is niks zonder zijn spelers! Letterelijk niet. Aan jou de taak om een tabel aan de database toe te voegen genaamd 'speler'.

  • Een speler heeft een naam
  • Een speler heeft een gamertag
  • Een speler heeft een ranking
  • Een speler heeft een preffered lane (top, bot, sup, mid of jung)

Een team kan meerdere spelers hebben (1 op veel)

Haal alle spelers op bij elk team en laat deze zien in de gui.

Technische eisen

  • De speles moeten opgehaald worden via entity framework
  • De vulling van de database moet via een SQL post deployment script
  • De spelers moeten worden gesorteerd op achternaam
