Start project

We gaan aan de slag met het ORM Entity Framework. Als je er even niet uit komt lees dan de bijbehorende theorie goed door! We gaan gebruik maken van de volgende 3 projecten:

  • Entities.Database - Deze gaan we op onze lokale server deployen
  • Entities.Domain - Hier gaan we onze entities genereren
  • Entities.ConsoleApp - Hier gaan we onze db context benaderen

Voordat we beginnen

Deploy het database project op je lokale server. Entities.Database - Rechter muisknop - Publish. Als je even niet meer weet hoe je lokale server heet, kijk dan in de SQL Management studio tool. Vaak is het [computernaam]\\SQLEXPRESS.

En dan...

Nu is het tijd om een EDMX te genereren op basis van je eigen database. Als je dit echt netjes wilt doen genereer je je EDMX altijd in een apparte library genaamd 'Domain'. Weet je even niet meer hoe dit moet? Er staan mooie filmpjes en lesstof voor je klaar!

Als laatste!

Tijdens het aanmaken van je EDMX heeft Entity Framework een zogenaamde 'connection string' gegenereerd in de app.config file van het domein project. Copiër deze connection string naar de app.config van het console project.

Lijst van teacher

Nu we een EDMX en database hebben, kunnen we aan de slag!

Laat in de console applicatie een lijst van docenten (Teacher) zien. Maak gebruik van de context klasse (Het kan zijn dat hij bij jou anders heet dan in het voorbeeld).

Lukt het je ook om bij elke docent de bijbehorende workshops te weergeven?

Tip: Vergeet de 'include' methode niet.

Lijst van studenten

Maar er zijn nog helemaal geen studenten? Dat klopt! Voeg een nieuwe tabel toe aan de database genaamd 'Student'.

Zorg er nu voor dat het entity diagram in het Domain project geüpdate is. Laat nu ook in de console een lijst van studenten zien.

Tip: Test data nodig? Je kan zelf handmatig de database vullen, maar kijk eens naar de folder 'Scripts' in het Database project.

Studenten toevoegen

Als de console alle workshops en studenten heeft geprint moet de console vragen om input van de gebruiker.

Breid de console applicatie uit zodat de gebruiker een naam van een student kan intypen en deze toegevoegd wordt aan de database

Studenten in de les

Wel zo leuk toch? Anders staat de leraar er voor niks!

Maak in het Database project een koppel tabel tussen student en workshop. Vergeet de foreign keys niet aan te geven!

Update opnieuw het entity diagram in het Domain project. Kijk na het updaten goed wat het resultaat is. Zie je je koppel tabel terug? Geef als oefening de navigation property 'students' op de klasse workshop een andere naam. (bijvoorbeeld participants).

Laat nu per workshop ook alle participerende studenten zien.

Tip: vergeet de testdata niet!

Start project

Klaar met de workshop opdracht? Dan gaan we nu Entity Framework toepassen in een WPF project!

In het start project staan 3 projecten

  • LeagueOfEntities.Database
  • LeagueOfEntities.Domain
  • LeagueOfEntities.WPF

Voordat je kunt beginnen moet je de database deployen en controleren of de connection strings nog goed staan!

De applicatie is al zover af dat wanneer je deze start er een lijst van competities met elk 2 teams beschikbaar is.

De opdracht

Een team is niks zonder zijn spelers! Letterlijk niet. Aan jou de taak om een tabel aan de database toe te voegen genaamd 'speler'.

  • Een speler heeft een naam
  • Een speler heeft een gamertag
  • Een speler heeft een ranking
  • Een speler heeft een preferred lane (top, bot, sup, mid of jung)

Een team kan meerdere spelers hebben (1 op veel)

Haal alle spelers op bij elk team en laat deze zien in de gui.

Technische eisen

  • De spelers moeten opgehaald worden via entity framework
  • De vulling van de database moet via een SQL post deployment script
  • De spelers moeten worden gesorteerd op achternaam
